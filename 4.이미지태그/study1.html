이미지태그

이미지 파일을 연결해서 홈페이지에 보여주는 역할을 하는 태그

src속성에 이미지파일까지의 경로를 입력하여 작성한다

alt속성에는 이미지를 설명할 수 있는 대체텍스트를 제공해야 한다

모든 사람들이 자신의 환경에 구애받지않고 동등하게 홈페이지를 사용할 수 있어야 된다는 웹접근성을 지키기 위해 alt속성을 입력해야 한다

연결할 수 있는 이미지파일의 형태는 일반적인 이미지라면 대부분 가능하다. (jpg, png, gif, svg)

이미지 또한 글자와 같은 컨텐츠이므로, 링크를 걸어주고 싶을 땐 a태그로 감싸준다

이미지태그는 일반적인 텍스트 글자와 같은 컨텐츠로 취급된다. 우리가 글자를 읽고 해석한 뒤 제목이라는 문맥 혹은 설명이라는 문맥으로 결정하듯이 이미지도 전체적인 맥란 안에서 재목 또는 설명컨텐츠로 구분될 수 있다

비디오태그

비디오파일을 연결해서 화면에 보여주는 태그

비디오파일은 브라우저와 아직 호환성이 완벽하지 않기 때문에 mp4파일은 거의 모든 브라우저와 호환이 가능하여 브라우저에서 볼 수 있지만 다른 형태의 파일은 브라우저에서 호환되지않는 경우가 많다

속성명과 속성값이 같으면 속성값을 생략해도 된다

comtrols 속성 = 영상의 재생바를 만들어줌

autoplay 속성 = 자동재생 (muted 속성이 꼭 같이 들어가야함)

loop 속성 = 반복재생 (영상을 클릭했을 때 일시정지 되지 않게하려면 controls 속성을 지우면 됨)

poster 속성 = 썸네일 설정

비디오태그에 마침태그가 있는 이유는 자식태그로 source태그를 구성하여 여러가지 형태의 비디오 타입을 수용할 수 있게 하기 위해서다

비디오 태그는 자식태그 source태그를 위에서 아래로 탐색해나가면서 해당하는 브라우저와 호환되는 영상이라면 바로 플레이한다

유튜브영상 홈페이지에 사용하기

유튜브에서 영상 우클릭 -> 소스코드 복사하기 -> 붙여넣기 를 하면 홈페이지에서 유튜브 영상을 재생할 수 있다
